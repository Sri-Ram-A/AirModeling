Circular Interpolation Visualizer

This demo shows why wind direction and other circular variables must not be interpolated as ordinary numbers. The left side computes the interpolation directly on the circle using sin/cos. The right side visualizes the vectors, the midpoint, and the imputed values across the gap.

The key idea is simple: a direction is not a line on a number axis. It is a point on a circle. So the safe workflow is: angle → radians → unit vector → interpolate X/Y → reconstruct with atan2 → normalize to 0–360°.
